                                    Marco Jakob Mod George Seese 4 years ago

                                    to question1: I know it's a bit strange to put a class named "SometingController" into the view package but in that case it's actually ok. The fxml files and their corresponding controller classes are closley tied to each other (that's why I call them view-controllers). So normally, I'd try to keep as much logic away from those view-controllers and just use them to have a high-level access to the view. Then, the real business logic goes into a separate package.
                                    Our AddressApp does not contain a lot of business logic and most of it is in MainApp.

                                    I still think SceneBuilder should allow having a (view-)controller class in a separate package so I reported it as a bug: https://javafx-jira.kenai.c... (unfortunately, you need to create a free login to see the bug report)

                                    to question2: I originally wanted to do that but this caused problems with SceneBuilder 1.1 (it allowed view-controllers to only be in parent packages...). Now I could change it but I will probably still keep it that way for now. One could argue that the MainApp is the high-level entry point and passes responsibility down to its subpackages....

                                                                                Can confirm for Eclipse "IllegalStateException" (in relation to path of FXML files)

                                                                                the Class named "MainApp" is called FIRST

                                                                                as such, it must be placed FIRST in the controller package in the heirarchy (a.k.a. higher in the order level heirarchy).

                                                                                for example, the following packages should contain the following files...

                                                                                gui

                                                                                MainApp.java

                                                                                gui.controller

                                                                                Main.java

                                                                                gui.model

                                                                                (currently empty in tutorial part 1)

                                                                                gui.view

                                                                                RootLayout.xml

                                                                                PersonOverview.xml

                                                                                ---

                                                                                then, you can call the FXML files from MainApp.java in the manner as proscribed by the awesome tutorial creator as such...

                                                                                "view/RootLayout.fxml"

                                                                                and

                                                                                "view/PersonOverview.fxml"
