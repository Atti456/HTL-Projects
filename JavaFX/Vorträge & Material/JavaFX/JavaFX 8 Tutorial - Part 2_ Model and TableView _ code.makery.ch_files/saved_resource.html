                                                      regarding the section...

                                                      "The PersonOverviewController

                                                      I found an extremely simple work-around in eclipse (as this bothered me to no end) that allows you to maintain the integrity of the MVC design pattern, and DOES NOT force you to place the controller class in the view package.

                                                      You may place the controller file in a separate package, that is... the controller class proper and the associated FXML file WILL read the controller class IF you open the FXML (not in SceneBuilder but) in eclipse itself. Simply right click the desired FXML and click "Open" (not "Open with SceneBuilder"), and it'll open in the IDE

                                                      near the top of the file, you'll find the following line...

                                                      <anchorpane prefheight="400.0" prefwidth="600.0" xmlns="http://javafx.com/javafx/8" xmlns:fx="http://javafx.com/fxml/1" fx:controller="CHANGE THIS">

                                                      simply "CHANGE THIS" to the path proper for your desired controller file location and save file; then SceneBuilder will now acknowledge the path to it's controller OUTSIDE of its own designated view package.

                                                      Can't tell you how much this urked me. Maintain the integrity of the MVC Design Pattern!!! lol :)

                                                                        Marco Jakob Mod Allen_3 • 4 years ago

                                                                        I think your description is correct: In JavaFX you are creating a scene graph. The scene graph can be created or manipulated both from FXML or from Java. So, FXML maps directly to the Java classes. The scene graph is like the DOM in web develpment, you can create the DOM tree declaratively from HTML but also create and/or manipulate it from JavaScript.

                                                                        Now, to your question. The logical flow is as follows:

                                                                        1. The fxml is loaded with an FXMLLoader.
                                                                        2. This creates the scene graph.
                                                                        3. If the fxml contains a reference to a controller class, this class is instantiated.
                                                                        4. If the instantiated controller contains fxml fields they are filled with a reference to the corresponding object in the scene graph. Note: we never instantiate the controller, it is always automatically instantiated by the FXMLLoader. We then ask the FXMLLoader with the method getController() for the reference to the instantiated controller.

                                                                                                                          Marco Jakob Mod Cheng • 4 years ago

                                                                                                                          > I don't see any object of type MainApp being instantiated anywhere

                                                                                                                          There is a special "mechanism" for starting JavaFX application. The MainApp extends the Application class and by callin launch(...) in the main-method the MainApp is instantiated, the constructor gets called and then the start-method gets called.

                                                                                                                          > is it possible to outsource the creation of objects of type Person and the OberservableList to a different class

                                                                                                                          Yes but things might get a bit more complicated with object references beeing passed around. Because you'll need access to that new class from the controller classes. One might try using the Singleton pattern for that or a dependency injection mechanism (see afterburner.fx) but this is quite advanced!
