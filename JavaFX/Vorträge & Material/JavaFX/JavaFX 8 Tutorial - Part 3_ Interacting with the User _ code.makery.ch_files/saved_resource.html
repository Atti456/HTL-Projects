                                                              Marco Jakob Mod Allen_3 4 years ago

                                                              > `LocalDate::from`, I haven't seen the syntax of double-colon before

                                                              It is a method reference which is part of the new Java 8. Well, it is quite complicated but here is how it works:

                                                              1. The DateTimeFormatter.parse method takes a TemporalQuery query as its second argument: See http://docs.oracle.com/java...

                                                              2. The TemporalQuery is a "Functional Interface" which means it only has one method: http://docs.oracle.com/java...

                                                              3. In Java 8, instead of providing a class implementing the TemporalQuery interface we can provide a method with the same signature as the single method in TemporalQuery. This can either be done with a Lambda Expression or by providing a method reference with the double-colon expression. It so happens that the LocalDate::from method is exactly what we need: http://docs.oracle.com/java...

                                                              > The other is about the Lambda Expression. I found that if we don't use Lambda Expression, just stay with the anonymous class, we must import the functional interface we implement, while we don't have to do so if we use Lambda Expression. Why is this?

                                                              I actually don't know.
