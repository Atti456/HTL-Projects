            Marco Jakob Mod George Seese 4 years ago

            Wow, congratulations. It takes quite some time to go trough the entire tutorial; you might be the first one - at least the first one leaving a comment! It's always great to hear that it is of help to other people.

            Yes, this AddressApp does pretty much the same as the JavaFX 2 AddressApp but in a newer and somewhat more elegant way.

            About Java 8 "benefits": I see two sides to Java 8. First, I'm quite excited about the new features. The included JavaFX 8 is great, I really like the Date and Time API and also (some of) the new language features. But I see a great problem with all those new language features. As it is really appealing for an experienced Java programmer it gets more difficult to understand for someone rather new to Java. Every new feature increases the complexity.

            A quick example: I can think of at least six possible ways to write a handler for a button click with Java 8 (and I'm sure there are more than that)! With Java 7 it was roughly half as many possibilities.

            Now, if someone who's learning Java is seeking a solution, it is really confusing to see that many different ways of achieving the same thing. I won't go too much further into that here (maybe in a later blog post, we'll see).

            But don't get discouraged. I think after completing this tutorial you should be able to read Java 8 code and mostly have a feeling about what it's doing. Don't worry too much about if you're implementing something in "old" Java style or in "new" Java 8 style - that doesn't matter much. But it is relevant that you can read other people's code.

            - Marco
