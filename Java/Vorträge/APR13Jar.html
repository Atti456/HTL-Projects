JAR (Java-Archive) Files

Java Archive

Wenn man mit der Entwicklung seiner Applikation fertig ist, müssen neu entwickelte Klassen und die von ihnen benötigten Resourcen zum Einsatz gebracht werden. Statt die Klassen in einzelnen Ordnern abzulegen (mit Pfaden entsprechend der Paket-Struktur), empfiehlt es sich, die Klassen in einer einzigen Datei zusammenzupacken, in einem sogenannten JAR-File.

JAR-Files

Ein JAR-File kann beliebige und auch beliebig viele Dateien aufnehmen. Sie dienen dazu, zusammengehörige class-Files und andere Dateien zusammen zu packen. Dies erfolgt im ZIP-File Format. Ein JAR-File enthält für die enthaltenen Dateien den relativen Pfad (der dem Paketnamen entspricht) und den Namen. Die Dateien werden komprimiert, diese kann aber auch unterdrückt werden.

JAR-Files erstellen

Für die Erstellung gibt es das Java Archive Tool. Entwicklungsumgebungen wie Eclipse bieten den Export als JAR-Archive an.

Java Archive Tool

Dieses Tool wird über die Kommandozeile bedient.

Bsp.:

Ein JAR-File erstellen (c reate):

jar cf anwedung.jar *.class *.gif *.au

Ein JAR-File ergänzen (u pdate):

jar uf anwedung.jar diese.class

Ein JAR-File anzeigen(t of contents):

jar tf anwedung.jar

Ein JAR-File auspacken(e xtract):

jar xf anwedung.jar

JAR-Files mit Eclipse

File - Export - JAR-File - Namen und Pfad angeben - Finish

Das JAR-File Manifest

Wenn ein JAR-File erstellt wird , erhält es automatisch ein Manifest. Dies ist eine spezielle Datei mit Pfad und Namen META-INF/MANIFEST.MF. Die Manifest-Datei enthält Meta-Informationen über das JAR-Archive und über einelne Dateien. Im Manifest können weitere Informationen aufgenommen werden.

Applikationen aus einem JAR-File starten

Befindet sich die zu startende Klasse in einem JAR-File, die nicht im Class-Path liegt, dann hat man zwei Möglichkeiten, die Anwendung mit dem java Kommando zu starten:

mit der Option -classpath oder mit der Option -jar.

java -classpath

java -classpath pfad\jar1.jar;pfad\jar2.jar package.Startklasse

java -jar

Für diese Option muss die Manifest-Datei ergänzt werden. Dazu legt man sich eine Text-Datei an (der Name sei mainClass.txt) Als Inhalt bekommt die Datei folgendes:

Main-Class: package.Classname

"Leerzeile"

Wichtig ist, dass eine Leerzeile folgt. Dann wird das Manifest im JAR-File folgendermassen geändert.

jar -ufm anwendung.jar mainClass.txt

Die Anwendung wird nun so gestartet:

java -jar anwendung.jar

Bei einigen Betriebssystemen (W2K, NT) wird damit eine Verknüpfung auf das JAR-File möglich, die ein Starten von Java-Programmen durch einen Doppelklick auf den JAR-File Namen erlauben.

JavaDoc

Allgemeines

JavaDoc ist ein im JDK enthaltenes Utility. Mit seiner Hilfe werden die im Sprachstandard bereits enthaltenen Dokumentationskommentare von Java eine HTML-Referenz umgesetzt. Voraussetzung dazu ist "Disziplin" und Tool-Unterstützung.

Java-Kommentarsyntax

Ein JavaDoc-Kommentar wird immer mit /** eingeleitet und mit */ abgeschlossen

@see ref - fügt einen Verweis ein. Es gibt dazu unterschiedliche Varianten dazu.

  • @see classname
  • @see package.classname
  • @see package.classname#method

@version text - Fügt einen Versionseintrag ein. JavaDoc ignoriert diese Einträge solange nicht die Kommandozeilenoption -version angegeben ist.

@author text - Fügt einen Authoreintrag ein. JavaDoc ignoriert auch diesen Eintrag, solange nicht die Kommandozeilenoption -author angegeben ist.

@param param-name description - Dient der Beschreibung von Mehtoden Paramentern.

@return description - Dokumentation von Rückgabewerten.

@exception package.classname description - Fügt eine Beschreibung der geworfenen Exception ein.

/**

*

* Dieser Text erscheint auch in der Kurzfassung.

*

* Weitere Zeilen erscheinen nur in der ausführlichen Darstellung.

*

* @return java.lang.String

* @param text java.lang.String

* @param count int

* @see javadoc.JavaDocReal

*/

String document(String text, int count);

JavaDoc und Eclipse

In Eclipse markiert man mit der rechten Maustaste die Klasse die dokumentiert werden soll und wählt dann Export aus. Im Dialog wählt man den JavaDoc aus und gibt anschließend den Pfad zum Programm javadoc.exe an (Javadoc Command) und den Pfad und Dateinamen für das HTML Dokumentationsfile. Beim drücken von Finish wird die Dokumentation erstellt.
