Klassen-Attribute und Klassen-Methoden

In einer Klasse kann man auch Attribute und Methoden definieren, die sich nicht auf die Instanzen der Klasse beziehen, sondern auf die Klasse selbst.

Man spricht dann von

  • Klassen-Attributen und
  • Klassen-Methoden.

Klassen-Attribute und Klassen-Methoden erhalten bei ihrer Deklaration den Modifizierer static.

Wir haben damit zwei unterschiedliche Sätze von Attributen und Methoden. Um für Klarheit zu sorgen, spricht man von den Attributen und Methoden, die sich auf die Instanzen der Klasse beziehen gezielt von

  • Instanz-Attributen und
  • Instanz-Methoden.

Beispiel anhand der Klasse Konto

Die Klasse Konto definiert den Zinssatz zentral für alle Objekte der Klasse, d.h. das Attribut ist kein Instanz-Attribut sondern ein Klassen-Attribut. Auch ein solches Klassen-Attribut wird versteckt. Es werden entsprechende Klassen-Methoden zur Bearbeitung angeboten.

class Konto

{

             static private double zinssatz =2.5;

             static public void setzeZinssatz (double zs)

            {

                       zinssatz = zs;

            }

            static public double gibZinssatz()

            {

                       return zinssatz;

            }

 }

Innerhalb einer Klassen-Methode und einer Instanz-Methode  kann man auf die Klassen-Attribute und Klassen-Methoden der eigenen Klasse durch Nennung des Namens zugreifen.

Auf die Klassen-Attribute und Klassen-Methoden einer anderen Klasse greift man mit dem Zugriffsoperator „ .“ zu.

Zusammenfassung der Zugriffsregeln

In einer Instanz-Methode hat man Zugriff auf

  • die Attribute und Methoden des aktuellen Objekt,
  • die Attribute und Methoden von einem anderen Objekt des Typs der Klasse,
  • die Klassen-Attribute und Klassen-Methoden.

In einer Klassen-Methode hat man Zugriff auf

  • die Klassen-Attribute und Klassen-Methoden,
  • die Attribute und Methoden von einem Objekt des Typs der Klasse, wenn die Methode eine Referenz auf das Objekt besitzt.

Sichtbarkeit (Zugreifbarkeit)

Auch die Klassen-Attribute und Klassen-Methoden können mit den Modifizierern private und public versteckt bzw. offengelegt werden.

final - Klassen-Attribute

Klassen-Attribute können ebenfalls mit dem Modifizerer „final“ versehen werden.

final - Attribute haben i. A. einen Initialisierer, der den Wert des Attributs definiert.

class WichtigeKonstanten

{

          ........

          public final static double PI = 3.1416;

          ........

 }

Haben final - Klassen-Attribute keinen Initialisierer, dann müssen sie in einem statischen Initialisierungsblock initialisiert werden. Dieser befindet sich im Rumpf der Klasse und hat die Form:

class SinusKurve

{

         private static final double[] sinus;

         ..........

         static  

         {

                 sinus = new double [200];

                 for (int i = 0; i < sinus.length; i++)

                 {

                        sinus[i] = Math.sin((double) i);

                 }

 }

Eine Klasse kann mehrere statische Initialisierungsblöcke haben. Für jede Klasse werden zum Zeitpunkt ihres Ladens in die virtuelle Maschine ihre statischen Initialisierungsblöcke durchlaufen. Ein Initialisierungsblock kann z.B. dazu verwendet werden, Klassen-Attribute mit Werten zu versehen.

Notation in der UML

Klassen - Attribute und Klassen - Methoden werden unterstrichen. Die Sichtbarkeitszeichen sind dieselben wie bei den Instanz-Attributen und Instanz-Methoden.
