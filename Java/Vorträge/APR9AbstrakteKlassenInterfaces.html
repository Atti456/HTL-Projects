Abstrakte Klassen und Interfaces

Klassen ohne Objekte die nur Superklasse sein wollen

In vielen Fällen sind Klassen notwendig, von denen keine Objekte erzeugt werden sollen. Solche Klassen erklärt man zu abstrakte Klassen. Eine abstrakte Klasse hat i. A. den Zweck, als Superklasse eine grundlegende Funktionalität für Subklassen anzubieten.

Zusätzlich kann eine abstrakte Klasse Methoden deklarieren, die keine Implementierung haben. Solche Methoden nennt man abstrakte Methoden. Subklassen einer abstrakten Klasse bieten eine konkrete Implementierung der in der Superklasse abstrakt deklarierten Methoden an.

Eine nicht abstrakte Subklasse einer abstrakten Klasse muss die in der Superklasse als abstrakt deklarierten Methoden implementieren. Die Subklasse muss das nicht wenn sie selbst abstrakt ist.

Abstrakte Klassen und abstrakte Methoden

Eine Klasse wird als abstrakte Klasse deklariert, indem man ihr den Modifizierer abstract gibt.

public abstract class GeomObjekt

{...}

Von einer abstrakten Klasse können keine Objekte erzeugt werden.

Eine Methode einer abstrakten Klasse, deren Implementierung einer Subklasse überlassen wird, hat den Modifizeierer abstract.

public abstract void transformiere(...)

{...}

Referenz Variablen vom Typ einer abstrakten Klasse

Auch wenn von einer abstrakten Klasse keine Objekte erzeugt werden können, kann man doch Referenz-Variablen vom Typ der Klasse deklarieren.

GeomObjekt go = null;

Eine Refernz-Variable vom Typ einer abstrakten Klasse kann nur Referenzen auf Instanzen von Subklassen der abstrakten Klasse aufnehmen.

go = new Kreis ( ...);

Notation in der UML

Abstrakte Klassen und Methoden werden in der UML rekursiv geschrieben.

Interfaces

Ein Interface ist mit einer abstrakten Klasse vergleichbar, die ausschließlich public und abstract Methoden enthält.

public interface Druckbar

{

Drucksatz gibDruckAufbereitet( );

}

Interface Methoden sind immer implizit public und abstract.

Von einem Interface können keine Objekte erzeugt werden.

Interfaces werden von Klassen implementiert.

public class Dokument implements Druckbar

{

.....

}

Referenz-Variablen vom Typ eines Interfaces

Auch wenn von einem Interface keine Objekte erzeugt werden können, kann man doch Referenz-Variablen vom Typ des Interfaces deklarieren.

Druckbar druckbareSache = null;

Eine Referenz-Variable vom Typ eines Interfaces kann eine Referenz auf ein Objekt aufnehmen, dessen Klasse das Interface implementiert.

Dokument dokum = new Dokument ( );

.....

druckbareSache = dokum;
