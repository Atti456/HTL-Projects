Ableitung und Vererbung

Ziel beim Ableiten und Vererben ist es Eigenschaften einer bereits vorhandenen Klasse zu nutzen. Eine Klasse wird von einer bestehenden Klasse abgeleitet. Die neue Klasse ist eine Spezialisierung der alten Klasse. Es soll Programmcode wiederverwendet werden.

Subklasse - Superklasse

Eine Klasse Geschäftskunde wird zur Subklasse der Klasse Kunde, wenn man in der Deklaration der Klasse Geschäftskunde die Klasse Kunde mit dem Schlüsselwort extends angibt. Die Klasse Kunde ist dann Superklasse der Klasse Geschäftskunde.

class Geschäftskunde extends Kunde

{

            ..........

 }

Es darf hinter extends nur eine Klasse spezifiziert werden.

class Viereck

{

            ..........

 }

 

class Rechteck extends Viereck

{

           ...........

}

 

class  Quadrat extends Rechteck

{

           ..........

}

Bezeichnungen

  • Viereck ist eine direkte Superklasse von Rechteck.
  • Viereck ist eine indirekte Superklasse von Quadrat.
  • Rechteck ist eine direkte Subklasse von Viereck.
  • Quadrat ist eine indirekte Subklasse von Viereck.
  • Quadrat erbt von Viereck indirekt durch Rechteck.

Was erbt die Subklasse

Unter einer Schnittstelle einer Klasse versteht man ihre von außen zugreifbaren Attribute und Methoden. Eine Subklasse erbt die Schnittstelle (sogenannter Kontrakt) ihrer Oberklasse. Public und Protected Elemente sind zugreifbar und Private Elemente sind nicht zugreifbar. Die Konstruktoren werden nicht verebt.

Die Klasse Object

Jede Klasse (außer der Klasse Object) ist implizit Subklasse der Klasse Object. Die Object Klasse ist die eine gemeinsame Wurzel der Java-Verebungshierachie. Wenn eine Klassendeklaration keine "extends" enthält, ergänzt Java implizit extends Object. Jede Klasse erbt implizit von Object u.a. die folgenen Methoden:

clone() – erzeugt eine identische Kopie des Objekts

equals() – prüft ob beide Objekte den selben Inhalt aufweisen. Unterscheide dazu „==“.

toString() - jede Klasse, welche toString() überschreibt, kann hier den relevanten Inhalt der Klasse in Textform ausgeben. Mit System.out.println(object); kann man den Inhalt eines Objektes ansehen.

Möglichkeiten durch Vererbung

Eine abgeleitete Klasse kann man dadurch:

Erweitern: Neue Elemente hinzufügen

Überschreiben: Geerbte Elemente verdecken, indem man sie durch eigene ersetzt.

Definieren: Eine Vorgabe realisieren.

Zugriff auf verdeckte Elemente der Superklasse

Auf verdeckte Instanz - Attribute und Instanz - Methoden greift man mittels

„super.element“

zu, wobei „element“ für das Attribut bzw. die Methode steht.

Auf verdeckte Klassen-Attribute und Klassen-Methoden greift man mittels

„name.element“

zu, wobei „name“ für den Klassennamen der Superklasse steht.

final - Methoden

Eine final - Methode kann in Subklassen nicht mehr überschrieben werden.

Konstruktoren

Konstruktoren werden nicht vererbt. Um den Konstruktor der Superklasse auszuführen, muss er im Konstruktor der Subklasse explizit aufgerufen werden. Der Aufruf des parameterlosen Konstruktors der Superklasse erfolgt mit super(). Bei Aufruf von Konstruktoren mit Parametern müssen die entsprechenden Argumente spezifiziert werden.

super();

super(mp);

Konstruktoraufruf in einem Konstruktor

In einem Konstruktor können Konstruktoren der eigenen Klasse und Konstruktoren der Superklasse aufgerufen werden. Mit super() wird ein Konstruktor der Superklasse aufgerufen, mit this() einer der eigenen Klasse.

Ein Konstruktor - Aufruf muss die erste Anweisung im Konstruktor sein. Weitere Konstruktor Aufrufe sind nicht erlaubt. Enthält ein Konstruktor als erste Anweisung keinen Konstruktor Aufruf, erfolgt implizit der Aufruf des parameterlosen Konstruktors der Superklasse.

Die Konstruktion von Objekten

Zuerst erfolgt die Initialisierung der Attribute mit Standard bzw. Initialisierungswerten. Danach wird der entsprechende Konstruktor aufgerufen.

Falls:

Die erste Anweisung super(...), Aufruf des Konstruktors der Superklasse.

Erste Anweisung this(...), Aufruf des entsprechenden Konstruktors der eigenen Klasse.

Andernfalls:

Implizite Ergänzung mit super( ), i.e., Aufruf des parameterlosen Konstruktors der Superklasse.

Typ-Kompatibilität

In einer Referenzvariablen können Referenzen gespeichert werden, deren Typ zum Typ des Objektes kompatibel ist. Alle von einer Klasse O direkt oder indirekt abgeleiteten Klassen definieren einen zum Typ O kompatiblen Typ.

Mit dem Operator instanceof kann man ein Objekt befragen, ob es zu einem bestimmten Typ kompatibel ist. Der Operator liefert den Wert true, wenn das Object zum spezifizierten Typ kompatibel ist, andernfalls false.

Bsp.:

Die Klasse GeomObjekt ist die Superklasse. Von dieser Klasse erbt Viereck und Kreis. Rechteck erbt von Viereck. Quadrat erbt von Rechteck.

GeomObjekt go = null;

go = new Kreis();

.....go instanceof Kreis .... //true

.....go instanceof GeomObjekt ... //true

.....go instanceof Viereck ... //false

go = new Rechteck();

..... go instanceof Viereck ... //true

..... go instanceof GeomObjekt ... //true

..... go instanceof Quadrat ... //false

..... go instanceof Kreis ... //false

Polymorphismus

Polymorph heißt allgemein vielgestaltig oder verschiedengestaltig. In der Informatik versteht man darunter, dass sich eine Operation je nach Kontext unterschiedlich verhalten kann. Man unterscheidet zwei Arten von Polymorphismus, den statischen und den dynamischen.

Vom statischen Polymorphismus spricht man im Zusammenhang mit dem Überladen/Überschreiben von Methoden.

Vom dynamischen Polymorphismus spricht man wenn sich verschiedene Klassen in einer Vererbungshierachie befinden und Methoden mit gleicher Signatur besitzen, welche die entsprechenden Methoden in der jeweiligen Oberklasse überschreiben.

Erst beim Aufruf einer Methode wird ermittelt, welche Methode in der Vererbungshierachie benutzt wird.
