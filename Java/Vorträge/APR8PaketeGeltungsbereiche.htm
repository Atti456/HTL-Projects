Pakete und Geltungsbereiche

Übersetzungseinheit

Eine Übersetzungseinheit umfasst den Quellcode, der vom Java Compiler in einem Übersetzungsvorgang übersetzt wird. In der Regel ist das genau eine Klassen- oder Interfacedeklaration. Dies entspricht dem .java File. Übersetzungseinheiten können auch mehrere Typdeklarationen enthalten, aber nur eine darf davon public sein.

Eine Übersetzungseinheit sollte nur eine Klasse enthalten. Der Name der Übersetzungseinheit sollte mit dem Namen der Klasse übereinstimmen. Bei einer public Klasse muss der Name der Übersetzungseinheit mit dem Namen der Klasse übereinstimmen.

Pakete

Klassen werden in Pakete organisiert. In einem Paket werden solche Klassen zusammengefasst, die eng zusammenarbeiten oder ein gemeinsames Thema haben. Mit einem Paket entsteht ein neuer Geltungsbereich für die in dem Paket enthaltenen Klassen. Innerhalb eines Pakets besteht Eindeutigkeit der Klassennamen.

Notation in der UML

Pakete werden als Karteikartenreiter dargestellt bzw. in Diagrammen ohne Paketdarstellung kann die Paketzugehörigkeit direkt in der Klasse per Doppelpunktnotation angegeben werden.

Bsp.:

GeometrischeFläche::Kreis

GeometrischeFläche::GefüllteFläche::GefüllterKreis

Erstellen von Paketen

Jede Übersetzungseinheit, genauer der Inhalt davon gehört genau zu einem Paket. In einer Übersetzungseinheit wird mit einer package - Deklaration spezifiziert, zu welchem Paket sie gehört. Ein Paket besteht also aus allen Übersetzungseinheiten, die den Namen des Pakets spezifizieren. Fehlt die package - Deklaration, dann gehört die Übersetzungseinheit zum so genannten namenlosen Standard-Paket ("Default - Package").

Jedes Paket hat einen Namen. Dieser besteht aus einer Folge von Bezeichnern, die durch einen Punkt miteinander verbunden sind.

Bsp.:

package hub.kurs.java.einstieg;

public class BildschirmAnzeige

{

....

}

Sichbarkeit

Innerhalb eines Pakets besteht "Sicht". Wenn eine Klasse des Pakets eine andere Klasse des Pakets benützen will, so kann sie das ohne weitere Vorkehrungen tun. Wenn eine Klasse außerhalb eines Pakets eine Klasse des Pakets benützen will, muss die Klasse innerhalb des Pakets public sein. Außerdem muss man das Paket, welche die public Klasse enthält importieren, indem entweder das ganze Paket importiert oder nur die gewünschte Klasse oder die Klasse voll qualifiziert im Programmcode anspricht.

Bsp.:

import hup.kurs.java.einsieg.*; // Alle Klassen werden importiert

import hup.kurs.java.einstieg.BildschirmAnzeige; // Nur die ein Klasse wird importiert

(new hup.kurs.java.einstieg.BildschirmAnzeige( )); // Vollqualifiziertes ansprechen im Sourcecode

Das Paket java.lang wird implizit importiert, dh. der Java Compiler fügt von sich aus die Anweisung import java.lang.*; ein.

Zwischen Paketnamen und dem Pfad der .java und .class Dateien besteht ein unmittelbarer zusammenhang. Ist v das aktuelle Verzeichnis und ist b1.b2. ... .bn der Paketname, dann muss es in v ein Unterverzeichnis geben mit dem relativen Pfad b1\b2\ .... bn geben, indem sich die .java Dateien befinden. Die .class Dateien werden im selben Verzeichnis abgelegt und dort ist auch der Paketname verzeichnet.

Geltungsbereiche

In Java werden Pakete, Klassen, Interfaces, Variablen, Methoden, lokale Variablen und Parameter durch eine Deklaration eingeführt. All diese Dinge erhalten dabei einen Namen. Unter dem Geltungsbereich versteht man den Java Code, in dem man den Namen ohne jeden Zusatz verwenden kann.

Der Geltungsbereich von Paketnamen hängt vom verwendeten Java System ab. Es muss sichergestellt sein, dass alle Pakete, die verwendet werden, im Geltungsbereich enthalten sind.

Der Geltungsbereich von Klassennamen ist das Paket, in dem sich die Klasse befindet. Dasselbe gilt für Interfacenamen.

Der Geltungsbereich eines Klassenelements ist der gesamte Rumpf der Klassendeklaration, unabhängig davon, ob es sich um ein geerbtes Element handelt oder nicht. Analoges gilt für Interfaces.

Der Geltungsbereich eines Methoden- oder Konstruktorparameters ist der gesamte Rumpf der Methoden - bzw. Konstruktordeklaration.

Der Geltungsbereich einer lokalen Variablen ist der Rest des Blocks, in dem die Deklaration enthalten ist.

Der Geltungsbereich einer lokalen Variablen, die in der for Laufbedingung deklariert ist, ist der Rest der for Anweisung.

Zugriffsrechte

Der Zugriff auf Klassen, Methoden, Variablen, etc. ist nur möglich, wenn man neben einem gültigen Namen auch die Zugriffsrechte besitzt. Die Verwendung von Namen wird vom Compiler einer Zugriffskontrolle unterzogen.

Zugriffsrechte werden mit den Modifizierer public, protected und private und durch die Standardeinstellung (kein Modifizierer) gesteuert.

Ob ein Paket zugreifbar ist, hängt vom verwendeten Java System ab. Die Zugriffsrechte für Pakete werden nicht durch Java Konstrukte gesteuert.

Eine Klasse oder ein Interface welches public deklariert ist, ist überall dort zugreifbar wo auch das Paket dem sie angehört zugreifbar ist. Eine Klasse mit dem private Modifizierer ist nicht sinnvoll und auch nicht möglich.p>

Elemente der Klasse sind nur zugreifbar wenn die Klasse zugreifbar ist.

public Elemente sind für jeden Code zugreifbar für den auch die Klasse zugreifbar ist.

private Elemente sind nur innerhalb der Klassendeklaration zugreifbar.

protected Elemente sind innerhalb des Paketes, dem die Klasse angehört und in Subklassen zugreifbar.

Elemente mit Standardzugriffsrechten sind innerhalb des Pakets, das die Deklaration enthält zugreifbar. In Subklassen sind sie nicht automatisch zugreifbar, es sei denn, die Subklasse liegt im selben Package.
