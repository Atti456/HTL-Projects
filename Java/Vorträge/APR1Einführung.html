JAVA

Entwicklungsgeschichte

1990 hatte der Software-Ingenieur J. Gosling begonnen, im Auftrag der Firma SUN unter dem Projektnamen OAK eine neue Objektorientierte Programmiersprache zu entwickeln. Die Vorgaben waren:

·        einfache Entwicklungsmöglichkeiten von Applikationen

·        Stabilität und Zuverlässigkeit von Programmen

·        Plattformunabhängigkeit

Als Syntaxbasis wurde ein abgespecktes C++ verwendet. Das OAK Projekt wurde 1994 auf Eis gelegt. Aus urheberrechtlichen Gründen wurde OAK in Java unbenannt.

Als SUN einen WWW Browser entwickelte wurde auf die Grundlagen von OAK zurückgegriffen. Die Möglichkeiten, die der Browser bot waren bahnbrechend, sodass SUN das Java-Projekt wieder  als Client-Server Technologie aufnahm.

Um Java populär zu machen, stellte SUN die Programmiersprache im Mai 1995 ins Internet. Dort gewann diese Sprache eine ungeheure Popularität, da Java neben separaten Programmen (Applikationen) auch die Ausführung eines so genannten Applets innnerhalb einer WWW-Seite erlaubt.

Java kurz vorgestellt

Java umfasst folgende Komponenten:

·        die Java Programmiersprache

·        die Java Virtuelle Maschine

·        die Java Plattform

Die Java Programmiersprache ist eine objektorientierte Programmiersprache. Sie verbindet Konzepte von Smalltalk und C++, hat aber nicht die Komplexität von C++ und weist auch eigene Konzepte auf.

Konzepte von Smalltalk:

·        Zugriff auf Objekte über Referenzen

·        Einfach-Vererbung

·        Compilation in Maschinen unabhängigen Code

·        Speicher Management mit Garbage Collection

Konzepte von C++:

·        Syntax der Anweisungen

·        Syntax der Klassenkonstruktion

·        Syntax des Umgangs mit Objekten

Eigene Konzepte:

·        Interfaces

Übersetzte Java Programme laufen in der Java Virtuellen Maschine (Java Virtual Machine, VM). Dazu müssen die Java Programme zuerst in den sogennanten Bytecode übersetzt werden.

Virtuelle Maschinen existieren für die verschiedensten Betriebsystem-Plattformen. Damit ist gewährleistet, dass ein auf einer beliebigen Betriebsystemplattform entwickeltes Programm unverändert auf allen anderen Betriebsystemplattformen läuft.

Programmentwicklung und Laufzeit

Die Java Plattform (Java Platform) besteht aus einer großen Menge vorgefertigter Klassen (Klassenbibliothek). Diese Klassen sind in sogennanten Packeten (Packages) organisiert.

Diese Pakete decken die unterschiedlichsten Bereiche ab, z.B.:

·        Strings

·        Collections

·        Prozesse und Threads

·        Eingabe/Ausgabe

·        Networking

·        graphische Oberfläche (Abstract Windows Toolkit (AWT), Swing, Java FX)

Die Java Plattform ist in das so genannte Java Development Kit (JDK) integriert. Dieses JDK wird von SUN kostenlos zur Verfügung gestellt und umfasst folgende Komponenten:

·        Compiler (javac)

·        komplette Java Plattform

·        VM

·        Utilities wie JavaDoc (javadoc), Builder Jar-Files Builder (jar)

·        Dokumentation in HTML Form
