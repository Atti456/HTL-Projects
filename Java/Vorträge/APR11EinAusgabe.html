Ein und Ausgabe

In-/Output in Java

Dienstklassen für die Ein- und Ausgabe finden sich im Paket java.io.* .

Darin finden sich :

  • Abbildung für Files
  • Directory Handling
  • Ein- und Ausgabeströme verschiedener Art
  • Exception-Klassen für spezielle IO-Fehler

Mit Datenströme geht man ganz "klassisch" um, nur sind Ströme Objekte:

  • Strom-Objekt öffnen,
  • schreiben/lesen als Methoden dieses Objekts,
  • Srom-Objekt schließen

Byte Streams und Character Streams

Es gibt Byte und Character Streams. Byte Streams sind sogenannte Input Streams bzw. Output Streams. Einheiten sind Bytes. Diese sind für Binär-Daten, Images und Sounds geeignet. Character Streams sind sogenannte Readers bzw. Writers. Einheiten sind hier zwei Bytes. Diese sind für textuelle Information geeignet, da alle Zeichen des Unicode Character Set verarbeitet werden können.

Andere Klassen in java.io

Neben den Klassen in den gezeigten Hierachien gibt es einige Klassen außerhalb dieser Hierachien, z.B.:

  • File: Abstraktion für Dateipfadnamen im lokalen Dateisystem
  • RandomAccessFile: Mechanismen zur Behandlung von Dateien als Ströme mit freier Zugriffswahl
  • StreamTokenizer: Zerlegt eine InputStream in Token. Wird oft benutzt, um Eingaben zu parsen, bzw. zu analysieren.

Der Exception-Handler ergreift die geeigneten Maßnahmen zur Behandlung der Ausnahmesituation.

Byte Streams

Wichtige Methoden von InputStream

int read( ) throws IOException - liefert ein Byte in einem int, -1 bei End of Stream.

int read(byte[] buf, int offset, int count) throws IOException - liest count Bytes in buf ein, -1 bei End of Stream

int read(byte[] buf) throws IOException - ruft folgendes auf: read(buf,0,buf.length)

void close( ) throws IOException - schließt den Strom

Wichtige Methoden von OutputStream

void write(int b) throws IOException - schreibt b als Byte

void write(byte[] buf, int offset, int count) throws IOException - schreibt einen Teil von buf

void write(byte[] buf) throws IOException - ruft folgendes auf: write(buf, 0, buf.length)

void flush( ) throws IOException - löst die Ausgabe auf das Medium aus

void close( ) - schließt den Strom

Herausgegriffen

FileInputStream und FileOutputStream liest bzw. schreibt File - Ströme.

ByteArrayInputStream und ByteArrayOutputStream verwendet ein Bytefeld als Ein- bzw. Ausgabestrom.

BufferedInput Stream und BuferedOutputStream gepufferte Datenströme lagern viele Zugriffe in den Speicher aus, damit nicht immer auf das File-System zugegriffen werden muss.

PipedInputStream und PipedOutputStream stellt eine Verbindung zwischen zwei Objekte her. Sie treten immer paarweise auf. Strom wird mit einem Eingabe- bzw. mit einem Ausgabestrom gekoppelt.

DataInputStream und DataOutputStream sind Ein- und Ausgabeströme für verschiedene Datentypen mit entsprechender Konvertierung.

Plattformunabhängige Ein- und Ausgabe

Die Properties aus der Klasse System sollten benutzt werden, um Datei-Ein-/Ausgabe zu realisieren, die wirklich plattformunabhängig ist.

String nl = System.getProperty("line.seperator");

String fs = System.getProperty("file.seperator"); // Properties holen

 

String filename = fs + "myfile"; // Filename zusammensetzen

 

FileOutputStream outputStream = new FileOutputStream(filename, true);

 

.........

 

String out = "Ausgabe " + nl; // Plattformunabhängigen Zeilentrenner verwenden

outputStream.write(out.getBytes());

 

Im Paket java.io gibt es zu viele Klassen, um sie einzeln vorzustellen. Für die meisten Anwendungen finden sich gezielt passende Ein- und Ausgabeklassen. Viele Methoden dieser Klassen zwingen den Anwender, ein Exception Handling durchzuführen.

Wichtige Methoden von Reader

int read( ) throws IOException - liefert ein Character in einem int, -1 bei End of Stream

int read(char[] buf, int offset, int count) throws IOException - liest Characters ein in buf, leifert -1 bei End of Stream

int read(char[] buf) throws IOException - ruft folgendes auf: read(buf, 0 , buf.length)

void close( ) throws IOException - schließt den Strom

Wichtige Mehtoden von Writer

void write(int b) throws IOException - schreibtb als Character

void write(char[] buf, int offset, int count) throws IOException - schreibt einen Teil des buf

void write(buf, 0 , buf.length) - ruft folgendes auf: write(buf, 0, buf.length)

void write(String str, int offset, int count) - schreibt einen Teil des str

void write(String str) throws IOException - ruft folgendes auf: write(str, 0 , buf.length)

void flush( ) throws IOException - löst Ausgabe auf das Medium aus

void close( ) throws IOException - schließt den Strom

Textdateien schreiben mit PrintWriter

FileOutputStream fos = new FileOutputStream("info.txt");

 

PrintWriter pw = new PrintWriter(fos);

 

pw.println("Hallo, wie gehts?);

PrintWriter stellt folgende Methoden zur Verfügung, ohne und mit New Line:

print(boolean b), println(boolean b)

print(char c), println(char c)

.......

print(Object obj), println(Object obj)

Textdateien lesen mit FileReader

FileReader fr = new FileReader("info.txt");

 

BufferedReader br= new BufferedReader(fr);

 

br.readLine( );

BufferedReader stellt die Methode readLine( ) zur Verfügung.

Die Klasse File

Die Klasse File repräsentiert das Inteface um das File-System der zugrundeliegenden Plattform zu manipulieren. Mit der Klasse File können keine Manipulationen des Dateiinhalts durchgeführt werden. Hierzu gibt es die Klassen FileInputStream, FileOutputStream und RandomAccessFile. Ein File-Objekt repräsentiert eine Pfadnamen für eine Datei oder ein Verzeichnis.

File definiert plattformunabhängige Klassen-Konstante für den "plattformunabhängigen" Gebrauch.

Für Verzeichnisangaben:

public static final char seperatorChar

public static final String seperator

für '/' oder '\' für UNIX bzw. Windows

Für Pfadangaben:

public static final char pathSeperatorChar

public static final String pathSeperator

für ':' oder ';' für UNIX bzw. Windows

Die Klasse File besitzt verschiedene Konstruktoren:

File(String pathname) // pathname kann ein absoluter oder relativer Pfad sein

File(String directoryPath, String filename) // erzeugt ein File-Objekt für die angegebene Datei im angegebenen Verzeichnis.

Folgende Methoden zur Existentsprüfung,zum Verzeichnis auslesen und zur Dateimanipulation stehen zu Verfügung:

  • boolean exists( )
  • boolean isFile( )
  • boolean isDirectory( )
  • String[] list( )
  • File[] listFile( )
  • boolean createNewFile( ) throws IOException
  • boolean mkdir( )
  • boolean renameTo(File dest)
  • boolean delete( )

Eingabe über die Tastatur

Die Tastatureingabe wird über das InputStream-Objekt System.in zur Verfügung gestellt. Ein InputStream ist ein Byte-Stream. Man müsste gegebenfalls ein eigenes "Character-Encoding" durchführen beim Übergang zu Characters.

 

byte[] bytes = new byte[80];

System.in.read(bytes);

String s = new String(bytes); // Kein Encoding

String ls = System.getProperty("line.seperator");

s = s.substring(0, s.indexOf(ls);

 

Das Encoding kann man mit einem Objekt der Klasse InputStreamReader bekommen. Dies ist ein CharacterStream. Wenn man außerdem noch mit einem BufferedReader arbeitet, ist das zeilenweise Lesen verbunden mit einer Bufferung.

 

InputStreamReader isr = new InputStreamReader(System.in);

BufferedReader br = new BufferedReader(isr);

String s = br.readLine( );

 

Ausgabe auf den Bildschirm

Die Bildschrimausgabe wird über das PrintStream Objekt System.out zur Verfügung gestellt. Dies ist ein Byte Stream. PrintStream ist mittlerweile deprecated und durch die Klasse PrintWriter ersetzt worden.

PrintWriter out = new PrintWriter(System.out);

out.println("Hallo Ausgabe!");

Auf der Konsole werden bei der Ausgabe mit System.out die Umlaute nicht richtig ausgegeben, Abhilfe schafft.
