Wichtige Collections

Sammlungen von Objekten

Wenn man mehrere Objekte z.B. in einem Array ablegt, dann kann man das Array als eine Sammlung von Objekten ansehen. Das typische dabei ist, dass die Sammlung eine Einheit bildet in Form eines Objektes, das seinerseits Objekte aufnehmen und verwalten kann.

Arrays sind in Java zwar in die Sprache integriert, sind aber in ihrer Funktionalität relativ beschränkt. Deshalb gibt es in Java neben den Arrays eine Serie von Klassen, die es erlauben Sammlungen von Objekten zu erstellen. Solche Objekte nennt man in Java Collections. Eine Collection verfügt generell über Methoden, um Objekte einzufügen und wieder zu entfernen, und über Methoden die Abfragen zulassen. Java bietet Collections in einem kompletten Framework an.

Das Collections Framework befindet sich in dem Paket java.util. Es besteht aus einer Reihe von Interfaces und einigen Klassen, die diese Interfaces implementieren. Je nach Aufgabenstellung wird man sich für den einen oder anderen Typ entscheiden. Wenn man den vollen Umfang des Frameworks nutzen will muß man dieses genau kennen.

Der Inhaltstyp von Collections

Der Inhaltstyp von Collections ist immer vom Typ Object. Wenn man ein Objekt in eine Collection einfügt, dann liefert man es unter seinem Typ ab, da alle Typen Sub-Typen von Object sind. Bei Abfragen, d.h. wenn man ein Objekt aus einer Collection bezieht, ist das anders. In der Regel muss erst wieder der korrekte Typ durch Casting hergestellt werden.

Bsp.: MeinTyp omt = (MeinTyp)liste.get(5);

Da in Collections nur Objekte abgelegt werden können, müssen die elementaren Datentypen in ein entsprechendes Wrapper-Objekt eingehüllt werden. Zu jedem elementaren Datentyp existiert eine Wrapper-Klasse.

HashSet

Die Klasse HashSet implementiert das Interface Set. Klassen, die das Interface Set implementieren, nennt man kurz Sets (Mengen). Sets lassen keine Duplikate zu. Ein HashSet verwaltet seinen Inhalt mit Hilfe einer Hash-Tabelle. Ein HashSet besitzt eine initial capacity und einen sogenannten load factor. Die capacity ist die Zahl der möglichen Einträge und die initial capacity ist die Kapazität unmittelbar nach der Erzeugung. Der load factor ist die Auslastung. Wird diese überschritten wird die Hash-Tabelle automatisch vergrößert, Standardwert dafür ist 0.75.

HashSet Konstruktoren

HashSet( ) // leerer HashSet mit Standard Kapazität und Standard Load Faktor

HashSet(int capacity) // HashSet mit Anfangskapazität und Standard Load Faktor.

HashSet(int capacity, float loadfactor) // HashSet mit Anfangskapazität und Load Faktor

HashSet(Collection c) // HashSet mit Anfangsinhalt

HashSet Methoden

add(Object o) // ein Objekt hinzufügen

addAll(Collection c) // eine Sammlung hinzufügen

boolean remove(Object o) // ein Objekt gezielt löschen

boolean remove(Collection c) // eine Sammlung gezielt löschen

clear( ) // den gesamten Inhalt löschen

int size( ) // Anzahl der Elemente erfragen

boolean isEmpty( ) // ist der HahSet leer?

boolean contains(Object o) // ist ein bestimmtes Objekt enthalten

boolean contains(Collection c) // ist eine bestimmte Collection enthalten

Iterator iterator( ) // Alle Elemente in einem Iterator auflisten

Object[ ] toArray( ) // Umsetzen in ein Array

Object[ ] toArray(Object[ ] a) // Umsetzen in ein spezifiziertes Array, die Methode gibt ein Array vom Typ a zurück. Die Elemente werden direkt in a geschrieben, falls sie dort Platz finden, ansonsten wird ein neues Array erzeugt und zurückgegeben. Sollte a größer sein als das Set Objekt enthält, dann enthalten die restlichen Komponenten in a null.

ArrayList

Eine ArrayList ist ein dynamisch wachsendes Array. Eine ArrayList bietet mächtigere Konstrukte als herkömmliche Arrays.

  • Einfügen an belieberger Stelle
  • Löschen an beliebiger Stelle mit nachrücken
  • Automatisches wachsen bei Bedarf

Auf die Elemente kann mit einem Index zugegriffen werden. Die Indizes laufen ab 0 wie bei den Arrays. Eine ArrayList ist dann sinnvoll, wenn genaue Angaben über die Größe eines Arrays fehlen oder die Größe beliebig ist.

ArrayList Konstruktoren

ArrayList( ) // leere ArrayList mit Standard Kapazität

ArrayList(int capacity) // ArrayList mit Anfangskapazität

ArrayList Methoden

add(Object o) // ein Objekt am Ende hinzufügen

add(int index, Object o) // Einfügen an der Stelle index

int set(int index, Object o) // Das Element an der Stelle index ersetzen

remove(int index) // Das Element an der Stelle index ersetzen

clear( ) // Alle Elemente löschen

Object get(int index) // Das Objekt an der Stelle index holen

int indexOf(Object o) // Nach dem ersten Vorkommen eines Objektes suchen

int lastIndexOf(Object o) // Nach dem letzten Vorkommen suchen

ListIterator listIterator( ) // Alle Elemente in einem ListIterator auflisten

int size( ) // liefert die Anzahl der Elemente

boolean isEmpty( ) // Leer oder nicht leer?

boolean contains(Object o) // Fragen, ob das spezifizierte Objekt enthalten ist

Object[ ] toArray( ) // Umsetzen in ein Array

Object[ ] toArray(Object[ ] a) // Umsetzen in ein spezifiziertes Array

HashMap

Eine HashMap ist eine auf einer Hash-Table basierende Map. Die Klasse HashMap implementiert das Interface Map. Eine Map bildet Schlüssel-Objekte (key) auf Wert-Objekte (value) ab, enthält also eine Kollektion von Key-Value-Paaren. Ein Schlüssel kann nur einmal vertreten sein.

Eine HashMap besitzt eine initial capacity und eine load factor, siehe unter HashSet.

HashMap Konstruktoren

HashMap( ) // leere HashMap mit Standard Kapazität

HashMap(int capacity) // HashMap mit Anfangskapazität

HashMap(int capacity, float loadfactor) // HashMap mit Anfangskapazität und Load Faktor

HashMap(Map m) // HashMap mit Anfangsinhalt

HashMap Methoden

Object put(Object key,Object value) // Hinzufügen eines Objektes, gibt eventuell ein unter gleichen Schlüssel vorhandenes Objekt zurück

Object get(Object key) // Holen des Objekts das mit key verbunden ist

Object remove(Object key) // gibt das gelöschte Objekt zurück

clear( ) // löscht die gesamte HashMap

Set keySet( ) // liefert alle Schlüssel zurück

Collection values( ) // liefert alle Werte zurück

Auf die zurückgelieferten Objekte kann man die Methode iterator( ) anwenden, die ein Iterator Objekt liefert.

boolean containsKey(Object key) // prüft ob Schlüssel vorhanden ist

boolean containsValue(Object value) // prüft ob Wert vorhanden ist

int size( ) // liefert die Anzahl der Elemente

boolean isEmpty( ) // Leer oder nicht leer?

Schlüssel müssen eindeutig sein, sonst wird ein vorhandenes Element entfernt und durch ein neues ersetzt. Values dürfen mehrfach vorkommen.

Iteratoren

Ein Iterator ist ein Objekt einer Klasse, die das Interface java.util.Iterator implementiert. Ein Iterator dient dazu, den Inhalt einer Collection in einfacher Form durchlaufen zu können, z.B. im Zusammenhang mit

  • Ausgabe
  • Suchfunktion
  • Umwandlungen

Collection Objekte und Set Objekte bieten die Methode iterator( ) an , die ein Iterator Objekt liefert.

Das Iterator Interface (Methoden von Iterator)

boolean hasNext( ) // gibt true zurück, wenn noch Objekte vorhanden sind.

Object next( ) // gibt das nächste, anstehende Objekt zurück und setzt seinen internen Zähler auf das folgenden Element.

void remove( ) // entfernt das zuletzt erhaltene Element aus der Collection. Diese Methode ist optional, falls sie nicht implementiert ist wirft sie eine Exception.

HashMap hm = new HashMap( );

........

Collection coll = hm.values( );

Iterator it = coll.Iterator( );

while(it.hasNext( )) // solange noch Elemente vorhanden

{         

             Typ dieses = (Typ)it.next( ); // nächstes Element holen

             ................................

 }

ListIteratoren

Ein Iterator ist ein Objekt einer Klasse, die das Interface java.util.ListIterator implementiert. Ein ListIterator ist eine Hilfsstruktur, die z.B. von der Klasse ArrayList genutzt wird, um den Inhalt in einfacher Form durchlaufen zu können.

Das ListIterator Interface (Methoden von ListIterator)

boolean hasNext( ) // gibt true zurück, wenn noch Objekte vorhanden sind.

Object next( ) // gibt das nächste, anstehende Objekt zurück und setzt seinen internen Zähler auf das folgenden Element.

boolean hasprevious( ) // gibt true zurück, wenn sich vor dem augenblicklichen Element ein weiteres befindet.

Object previous( ) // gibt das Element zurück, das sich unmittelbar vor dem augenblicklichen Element befindet.

ArrayList al = new ArrayList( );

.........

ListIterator lit = al.ListIterator( );

while(lit.hasNext( )) // solange noch Elemente vorhanden

{         

             Typ dieses = (Typ)lit.next( ); // nächstes Element holen

             ................................

 }

In der Regel bekommt man List-Iteraoren als Resultat einer Collection Klasse, z.B. von der Methode listIterator( ) der Klasse ArrayList. Will man eigene Datenstrukturen als List-Iterator benutzen, muß man das Interface ListIterator implementieren.
