Übung 12

Aufgabe Wichtige Collections

Ausgangspunkt dieser Übung ist die Lösung der Übung 4.

In der Klasse Bestellung werden die Bestellpositionen-Objekte in einem Array gehalten. Ein Array an dieser Stelle limitiert die Anzahl der Bestellpositionen und eine Erweiterung müsste von Hand programmiert werden, was wir gemacht haben. Einfacher wird dies aber wenn man sich ein Collection-Objekt anlegt. Ersetzen sie das Array durch eine ArrayList.

Führen Sie die Klasse Kundenverzeichnis in die Anwendung ein.

Die Klasse Kundenverzeichnis:

private:

-verzeichnis:HashMap

public:

+Kundenverzeichnis(kunde: Kunde[ ])

+fuegeEin(k: Kunde)

+gibKunde(kundennummer:String):Kunde

+entferneKunde(kundennummer:String)

+alleKunden( ):Iterator

+alleKundennummern( ):Iterator

+zeigeVerzeichnis( )

Das Attribut verzeichnis verweist auf eine HashMap, in der alle Kunden geführt werden. Der Schlüssel (Key) in der HashMap ist die Kundennummer, die im Kunde-Objekt enthalten ist, der Wert (Value) das Kunde-Objekt selbst.

In der Mehtode fuegeEin( ) wird also zuerst die Kundennummer aus dem Kunde-Objekt geholt, um es dann als Key-Objekt beim Einbringen ( put( ) ) in die HashMap zu verwenden.

Der Methode EntferneKunde( ) wird eine Kundennummer zur Verfügung gestellt, um den Kunden aus dem Verzeichnis zu entfernen.

Die Methode alleKunden( ) liefert einen Iterator über alle Kunden-Objekte.

Die Methode alleKundennummern( ) leifert einen Iterator über alle Kundennummern.

Der Konstruktor dient dazu, schnell ein Kundenverzeichnis zu erstellen. Ihm wird ein Array mit Kunde-Objekte übergeben. Im Konstruktor werden diese Objekte mit der Methode fuegeEin( ) im Verzeichnis untergebracht.

Erstellen Sie ein Testprogramm, das ein Kundenverzeichnis erstellt und anzeigt.
